Features

ActorDB is strongly consistent, distributed and horizontally scalable.

Unlike traditional monolithic databases, ActorDB is made out of any number of independent and concurrent SQL databases called actors.

Scalable

ActorDB can run on a single server or hundreds. It can store gigabytes or petabytes.

Features

Full relational database within an actor. Queries and transactions over multiple actors. Redundant. Massively concurrent. No single point of failure. ACID. Connectable over MySQL protocol.

Easy to use

Works with default MySQL drivers of your platform. Simple configuration and administration.
